# Your phone does not know what a photo is


I made this tool because of a question a student asked that I could not answer cleanly on the spot: "If the computer stores my photo, what is it actually storing?" We talk about the "digital" world as if it floats free of the physical one, but a computer does not understand numbers, text, or pictures in any direct sense. It understands voltage. Everything else is a convention we stacked on top of voltage so that humans could get useful work out of it. Tracing that stack from a single electrical state up to a full-color photo is one of the more satisfying things to show a student, because every step is concrete and nothing has to be taken on faith.



## What a bit really is

A **bit** (short for binary digit) is the smallest unit of information a computer stores, and it is a physical state, not an abstract idea. Inside the silicon of a processor sit billions of microscopic switches called transistors. When a transistor holds a high voltage, on the order of a volt in many chips, we agree to read it as a **1**. When the voltage drops near zero, we read it as a **0**. That is the whole alphabet: two states, nothing in between. That simplicity is exactly why it is so reliable. A circuit only has to tell "clearly on" from "clearly off" instead of measuring a precise level, so a little electrical noise on a cheap, shared machine will not flip a 1 into a 0.

That two-state design is also why files look enormous when you count them in bits. A 10-megabyte song is roughly 83,886,080 separate high-or-low states, each one a transistor pushed on or off. The number is huge because each bit, on its own, carries so little.

## Grouping bits into bytes

Reading tens of millions of individual 1s and 0s is hopeless for a person, so bits get grouped. The standard cluster is 8 bits, called a **byte**. Eight switches, each on or off, give $2^8 = 256$ distinct patterns, running from `00000000` to `11111111`. Those 256 patterns are enough to cover the basic Latin alphabet, the digits, and punctuation, which is how a byte became the common unit for a single character of text.

Each position in a byte carries a fixed value, a power of two, and the number a pattern stands for is just the sum of the place values that are switched on. Those place values double as you move from right to left:

Turn on every position and you add $128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255$, which is why the largest value a single byte holds is 255, not 256. (256 is the *count* of patterns, once you include zero.) To write these patterns compactly, programmers use **hexadecimal**, base 16, which counts 0 through 9 and then A through F before rolling over. Two hex digits cover one byte exactly, so `00000000` is `00` and `11111111` is `FF`. Hex is not a different kind of number, just a shorter way to spell the same bits.

## Painting with bytes

So how does voltage become a photograph? Through **pixels** (picture elements). Modern screens use the RGB model, where every pixel is three tiny physical lights, one red, one green, one blue. The computer gives each light one byte to set its brightness, so a value of 0 means that light is off and 255 means it is at full blast. Mixing the three at different levels produces every color the pixel can show.

Three bytes per pixel is 24 bits, which is where the term "24-bit true color" comes from. Because each of the three channels has 256 settings, the total is $256 \times 256 \times 256 = 16{,}777{,}216$ distinct colors, more than the human eye can reliably tell apart. The tool makes this chain visible. You get three rows of eight switches, one row per color channel, and each row shows its own decimal and hex value as you flip bits. Beside them, a single pixel updates live to the color those three bytes describe, along with a bar for how bright each subpixel is and the final hex code. Hit Randomize a few times, or just toggle one bit in the red row and watch the color shift, and the jump from "voltage" to "image" stops feeling like magic and starts feeling like counting.
